CLAUDE.md — 규칙과 맥락의 뿌리

Claude가 프로젝트에 들어올 때 가장 먼저 읽는 파일입니다. 팀의 규칙서이자 온보딩 문서입니다.

BEFORE
보고서 어투
매번 "~합니다 쓰지 마"라고 설명
참조 문서
"이 주제는 OKR 문서 먼저 봐야 해"
이전 결정
"지난번에 뭐라고 결정했지?"
AFTER
보고서 어투
어투 규칙이 매 세션 자동 적용
참조 문서
트리거 키워드 → 관련 문서 자동 참조
이전 결정
과거 세션 자동 검색 → 맥락 복원
✎
산출물 규칙
어투, 문서 구조, 금지 사항을 한번 정의
⇄
자동 참조 규칙
키워드를 감지하면 관련 파일을 자동으로 로드
▶
세션 시작 프로토콜
주제만 말하면 이전 맥락이 올라옴
신입한테 온보딩 문서 주듯이, Claude한테도 CLAUDE.md를 주면 됩니다.
한번 써놓으면 매번 설명할 필요가 없습니다.
실제 프롬프트 예시
1
"우리 팀 CLAUDE.md를 만들어줘.
산출물 규칙: 어투는 보고서체(~함/~임), 코드 블록 금지하고 테이블로 대체, 문서 구조는 H1 제목 → 최종 결정 사항 → 본론 → 실행 계획 순서로 고정.
금지 사항: 에이전트 역할명 노출 금지, 불확실한 정보를 단정 금지.
이 규칙들을 CLAUDE.md에 정리해줘"
→ 산출물 규칙 초안이 나옵니다
2
"자동 참조 규칙을 추가해줘.
'OKR, 목표, KR' 키워드가 나오면 knowledge/policies/2026-OKR.md를 먼저 읽고 답할 것.
'브랜드, 슬로건, 톤앤매너' 키워드가 나오면 knowledge/policies/BRAND-GUIDELINES.md를 참조할 것.
이런 식으로 키워드 → 파일 매핑 테이블을 만들어서 CLAUDE.md에 넣어줘"
→ 트리거 키워드 기반 자동 참조 규칙이 추가됩니다
3
"세션 시작 프로토콜을 정의해줘.
1) 세션 시작 시 MEMORY.md에서 미해결 이슈와 진행 중 논의를 확인할 것.
2) 사용자가 주제를 말하면 SESSION-INDEX.md에서 관련 이전 세션을 찾아서 맥락을 복원할 것.
3) 이전 결정과 충돌하는 방향이 나오면 반드시 이전 결정을 언급하고 변경 이유를 확인할 것.
이 프로토콜을 CLAUDE.md에 넣어줘"
→ 세션 시작 프로토콜이 정의됩니다

BONUS

더 잘 쓰는 법 — Plan Mode와 단축키

Claude Code를 쓰다 보면 매번 yes/no를 눌러야 하는 게 번거롭습니다.
이걸 건너뛰는 방법이 있습니다.

Plan Mode — 먼저 계획을 세우고, 한번에 실행하기
Shift + Tab을 누르면 Plan Mode로 전환됩니다.
이 모드에서 Claude는 바로 실행하지 않고, 먼저 계획을 보여줍니다.
계획이 마음에 들면 승인 한 번으로 전체를 실행합니다.

큰 작업을 할 때 "하나씩 yes 누르는 것"을 없앨 수 있습니다.
Auto-accept — yes/no 없이 자동 실행
Shift + Tab을 여러 번 누르면 모드가 순환합니다:
Plan → Auto-accept → Default.

Auto-accept 모드에서는 Claude가 확인 없이 바로 실행합니다.
신뢰할 수 있는 반복 작업에 쓰면 됩니다.
권한 설정 — 특정 도구만 자동 허용
매번 모드를 바꾸지 않아도, 자주 쓰는 도구를 미리 허용해둘 수 있습니다.
/allowed-tools에서 설정하면 해당 도구는 확인 없이 실행됩니다.
파일 읽기, 검색 같은 안전한 작업을 허용해두면 편합니다.

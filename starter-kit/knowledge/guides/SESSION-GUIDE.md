# 팀 세션 사용 가이드

## 세션이란?

팀을 복제한 Claude와 함께 진행하는 **문제 해결 세션**입니다. 각 세션은 하나의 문제를 정의하고, 논의하며, 의사결정을 내리는 전체 과정을 의미합니다.

---

## 세션 시작하기

### 1. 세션 준비

init.md와 필요한 페르소나를 로드한 뒤, 논의할 주제를 제시합니다.

### 2. 필요한 역할 추가 로드

세션 진행 중 필요에 따라 추가 페르소나를 로드합니다:
- "[팀장-lite].md 로드해서 방향성 확인해줘"
- "[전문가].md 로드해서 상세 분석해줘"
- "devils-advocate.md 로드해서 리스크 체크해줘"

### 3. 세션 진행

팀장 페르소나를 중심으로:
문제 정의 → 데이터 검증 → 옵션 탐색 → 리스크 체크 → 의사결정 → 실행 계획

## 세션 종료 및 저장

### 세션 종료 시

"이 세션을 저장해줘. 주제: [세션 주제] / 핵심 결정: [결정 사항 요약]"

### 자동으로 생성되는 파일

세션 저장 시 sessions/ 폴더에 3가지 파일이 자동 생성됩니다:
- FULL-LOG.md — 전체 대화 로그
- DECISION-DOC.md — 의사결정 문서
- MEETING-NOTES.md — 회의록

**산출물 작성 규칙:** knowledge/guides/OUTPUT-RULES.md 참조

---

## 파일별 용도

### 1. FULL-LOG.md (전체 로그)
**활용:** 의사결정 배경 추적, 학습 및 회고, 유사 문제 발생 시 참고

### 2. DECISION-DOC.md (의사결정 문서)
**활용:** Confluence 업로드 (메인 문서), 전사 의사결정 기록, 실행 팀 가이드

### 3. MEETING-NOTES.md (회의록)
**활용:** 빠른 공유 (Slack, 이메일), 액션 아이템 체크, 다음 회의 준비

## 세션 타입별 권장 로드

- 필수: init.md, [팀장-lite].md

### 실무 검토
- 필수: init.md, [담당 전문가].md
- 추천: [관련 전문가].md

### 리스크 높은 의사결정
- 필수: init.md, [팀장].md, devils-advocate.md
- 추천: [관련 전문가].md

## 컨텍스트 효율화 전략

### 세션 브리프 패턴

새 세션에서 이전 논의를 이어갈 때, 풀로그 대신 세션 브리프(3~5줄 요약)를 첫 메시지로 입력하여 컨텍스트를 절약합니다.

| 복원 방식 | 예상 토큰 | 맥락 복원율 |
|----------|----------|------------|
| 풀 세션 로그 로드 | 3,000~10,000 | 100% |
| DECISION-DOC 로드 | 1,000~3,000 | 80% |
| 세션 브리프 | 100~200 | 60~70% |

**권장:** 세션 브리프로 시작 → 필요 시 DECISION-DOC 추가 로드

### 논의 단계별 세션 분리

복잡한 의사결정은 단계별로 세션을 분리하여 컨텍스트 소모를 관리합니다.

| 단계 | 세션 | 로드 파일 | 산출물 |
|------|------|----------|--------|
| 1. 문제정의 | 세션A | init + [팀장-lite] | 문제정의 1페이지 |
| 2. 옵션 탐색 | 세션B | init + [전문가-lite] + 세션A 산출물 | 옵션 비교표 |
| 3. 리스크 검토 | 세션C | init + devils-advocate + 세션B 산출물 | 리스크 평가 |
| 4. 최종 결정 | 세션D | init + [팀장-lite] + 세션A~C 요약 | DECISION-DOC |

## FAQ

**Q1. 모든 세션을 저장해야 하나요?**
A: 아니오. 중요한 의사결정이나 복잡한 논의만 저장하세요.

**Q2. 세션 중간에 저장할 수 있나요?**
A: 네. "지금까지 논의 내용 저장해줘"라고 요청하면 중간 저장 가능합니다.

**Q3. 세션 파일을 수정할 수 있나요?**
A: 네. 마크다운 파일이므로 자유롭게 수정 가능합니다.

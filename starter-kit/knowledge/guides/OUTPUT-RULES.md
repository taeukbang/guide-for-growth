# 산출물 작성 규칙

> 세션 문서 작성 시 반드시 준수. CLAUDE.md에서 "세션 저장" 요청 시 이 파일을 먼저 읽도록 지시됨.

---

## 공통 규칙

| 항목 | 규칙 |
|------|------|
| 어투 | 보고서체 사용: ~함, ~임, ~필요함 (NOT ~한다, ~이다, ~합니다, ~됩니다) |
| 에이전트 메타 정보 | 문서에 역할명(Growth Director 등), 리뷰 참여자 목록 미포함 |
| 담당자 표기 | 일반 직무 타이틀 사용 (Data Analyst, CRM 담당 등) |
| Confluence 호환 | 코드 블록 사용 금지 → 테이블, bold 텍스트로 대체 |
| 이모지 | 사용 금지 |

**어투 퀵 레퍼런스:**

| 올바른 어투 | 금지 어투 |
|------------|----------|
| ~함, ~임, ~필요함 | ~한다, ~이다, ~합니다, ~됩니다 |
| 분석 결과 X로 나타남 | 분석 결과 X로 나타납니다 |
| 실행 계획은 다음과 같음 | 실행 계획은 다음과 같습니다 |

## DECISION-DOC.md 구조 (순서 고정)

**H1 타이틀:** 구체적 주제명 사용 (NOT "의사결정 문서")

**메타데이터 블록 (H1 직후):**
- **날짜:** YYYY-MM-DD
- **주제:** 한 줄 요약
- **유형:** 지표 전략 / 캠페인 기획 / 조직 설계 / 정책 결정 등
- **배경:** 한 줄 컨텍스트

**필수 섹션 (순서 고정):**

1. **## 최종 결정 사항** — 문서 첫 실질 섹션. 핵심 결정을 bold 번호 리스트로 요약
2. **## 1. 배경 및 문제 정의** — 왜 이 논의가 필요했는지
3. **## 2~N. 본론** — 주제에 따라 자유 구성 (분석, 옵션 비교, 전략 등)
4. **## N+1. 실행 계획** — Phase별 테이블 (액션 | 담당 | 기한)
5. **## 변경 이력** — 테이블 (버전 | 날짜 | 변경 내용)
6. **푸터** — 다음 리뷰 예정, 담당자, 승인자

## MEETING-NOTES.md 구조 (순서 고정)

**H1 타이틀:** # [구체적 주제명] - 회의록

**메타데이터 블록 (H1 직후):**
- **날짜:** YYYY-MM-DD
- **참석:** 팀/조직 단위 (에이전트 역할명 미표기)
- **주제:** 한 줄 요약

**필수 섹션 (순서 고정, 각 섹션 사이 --- 구분선):**

1. **## 안건** — 1~2줄 요약
2. **## 주요 논의** — 번호 매긴 H3 소제목으로 구분
3. **## 결정 사항** — 반드시 테이블: | 번호 | 결정 | 비고 |
4. **## 액션 아이템** — 반드시 체크리스트: - [ ] 담당: 내용 (기한)
5. **## 다음 회의** — 일정, 참석자를 불릿으로 표기

**금지 형식:**
- 결정 사항을 번호 리스트, H3+불릿, 한줄 bold 등으로 작성 금지 → 반드시 테이블
- 액션 아이템을 테이블로 작성 금지 → 반드시 체크리스트

## 세션 저장 워크플로우

사용자가 "세션 저장해줘" 요청 시:

**Step 1: 폴더 생성**
- 경로: sessions/YYYY-MM-DD_topic-name/
- topic-name: 영문 소문자, 하이픈 구분

**Step 2: 3개 파일 작성**
- FULL-LOG.md — 전체 대화 로그
- DECISION-DOC.md — 의사결정 문서 (위 구조 준수)
- MEETING-NOTES.md — 회의록 요약 (위 구조 준수)

**Step 3: SESSION-INDEX.md 업데이트**
- 마스터 테이블에 새 세션 행 추가
- 태그 인덱스 업데이트
- 토픽 클러스터에 배치

**Step 4: 품질 검증 체크리스트**

DECISION-DOC 체크:
- H1이 구체적 주제명인가
- 최종 결정 사항이 첫 섹션인가
- 실행 계획이 Phase별 테이블인가
- 어투가 보고서체인가
- 코드 블록이 없는가

MEETING-NOTES 체크:
- 결정 사항이 테이블인가
- 액션 아이템이 체크리스트인가
- 어투가 보고서체인가
